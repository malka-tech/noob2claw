# Noob2Claw – Folge 12: Wiki per Vibe Coding entwickeln

In Folge 12 wird ein professionelles Wiki schrittweise mit gezielten Prompts aufgebaut. Der Schwerpunkt liegt auf einem kontrollierten Vibe-Coding-Workflow statt auf einem einzigen großen Entwicklungsauftrag.

# 🌐 Noob2Claw-Webseite

Weitere Informationen, Anleitungen und Neuigkeiten zum Projekt findet ihr auf [noob2claw.de](https://noob2claw.de/).

---

# Ziel der Folge

Das Wiki soll als strukturierte Wissensbasis für Benutzer und spätere KI-Agenten dienen. Die Entwicklung erfolgt in klar abgegrenzten Schritten, deren Ergebnisse jeweils im Browser, in der Datenbank und im Code geprüft werden.

---

# Vorgehensweise

```text
Ziel vorgeben
      │
      ▼
KI entwickelt einen Teilbereich
      │
      ▼
Ergebnis prüfen
      │
      ▼
Gezielt nachprompten
      │
      ▼
Nächsten Teilbereich umsetzen
```

---

# Was wird behandelt?

- schrittweiser Aufbau eines Wiki-Moduls
- klare und überprüfbare Beispielprompts
- Strukturierung von Artikeln und Kategorien
- Bearbeitung, Suche und Darstellung von Inhalten
- Datei- und Medienverwaltung
- Versionierung und Nachvollziehbarkeit
- Benutzerrechte und Sicherheitsprüfungen
- konsequente Kontrolle jedes Entwicklungsschritts

---

# Enthaltene Datei

- `0_beispielprompts.md` – aufeinander aufbauende Prompts für die vollständige Wiki-Entwicklung

# Ergebnis

Nach dieser Folge steht ein Wiki als zentrale Wissensbasis bereit. Gleichzeitig zeigt die Folge, wie größere Module durch kleine, kontrollierte und nachvollziehbare Vibe-Coding-Schritte entstehen.
